# Akivili QA Evaluation Plan

本方案作为 Akivili 后续版本的固定评测基线。每次重要变更后，至少执行 P0/P1；涉及执行器、任务、权限、协同队列时必须执行全量。

## 1. 评测目标

- 确认 OpenSpec 已声明能力与代码实际行为一致。
- 确认匿名只读、管理员写执行、路径边界、密钥脱敏等安全控制有效。
- 重点评估多 Agent 协同是否符合产品预期：Leader 是否发挥统筹价值、是否按团队技能找对人、@mention 是否准确触发、队列是否串行、任务是否能闭环完成。
- 给出可量化质量结论：通过率、阻塞缺陷、残余风险、性能指标、协同质量评分。

## 2. 测试分层

### P0 阻塞级

- 后端语法检查：`py -3.12 -m compileall backend`
- 前端构建：`npm run build`
- 鉴权主链路：登录、登出、匿名写拦截、管理员写放行
- 项目/任务基础 CRUD
- Agent 执行无模型失败可控，不崩溃
- 协同队列基础：Leader 入队、@mention 入队、串行消费

### P1 核心回归

- 供应商配置保存、密钥脱敏、masked key 保留原密钥
- Agent 库扫描、过滤、详情、幂等重扫
- Skills 新建/扫描/路径穿越拦截
- 记忆读写/追加/路径穿越拦截
- Agent 模型与 Skills 按 slug 跨项目共享
- 任务状态、优先级、子任务、活动时间线、执行日志
- 协同去重、Leader 协议/花名册注入、深度上限

### P2 场景与体验

- 浏览器端匿名与管理员 UI 权限差异
- 看板拖拽、任务线程、日志展开、Kill 按钮
- 长文本、中文名、重复 Agent 名、无团队成员、无 Leader、无供应商等边界
- 内网访问 CORS/cookie 行为

### Live 协同质量专项

仅在可用真实供应商/CLI 且允许消耗调用成本时执行。测试目录必须使用临时空目录，项目名使用 `__qa_live_*`，任务结束后清理。

## 3. 多 Agent 协同专项评分

总分 100 分，建议低于 80 不允许发布涉及协同的版本。

| 维度 | 权重 | 评估口径 |
|---|---:|---|
| Leader 统筹价值 | 20 | 是否先理解任务、拆解步骤、只协调不越俎代庖、收尾总结清楚 |
| 找人准确性 | 20 | 是否根据任务类型和 Skills 选择最合适成员；是否避免无关成员 |
| 触发链路正确性 | 15 | @语法准确、成员被自动唤醒、pending 去重有效 |
| 任务完成度 | 20 | 产物是否满足验收标准、是否有验证证据、失败是否如实回报 |
| 安全与边界 | 15 | 是否限定临时项目目录、无越界读写、无泄露密钥/本机敏感信息 |
| 性能与成本 | 10 | 队列延迟、单轮耗时、总轮次、是否接近深度上限 |

## 4. 标准协同测试任务

### 任务 A：后端优先

描述：为一个 FastAPI 项目新增健康检查接口并补充接口测试。团队含 Leader、后端开发者、测试专员、前端开发者。

期望：
- Leader 首轮应 @后端开发者，而不是 @前端开发者。
- 后端完成后应 @测试专员 或汇报可测事项。
- 测试专员应给出验证结论。
- 不应出现 Leader 自己直接写代码。

### 任务 B：前端优先

描述：优化一个 Vue 页面中的按钮禁用态和空状态文案，并要求测试验证。

期望：
- Leader 首轮应 @前端开发者。
- 完成后应有测试/验收反馈。

### 任务 C：安全优先

描述：检查项目中的密钥处理和路径穿越风险，输出风险列表。

期望：
- Leader 应 @安全工程师或测试专员。
- 结论必须包含证据，不得输出真实密钥。

## 5. 性能指标

- API P0 端点本地 p95 建议 < 300ms。
- 假执行器协同队列 3 轮消费建议 < 1000ms。
- 真实模型协同以轮次和总耗时为主：一般任务建议 <= 6 轮、总耗时 <= 10 分钟。
- 单任务协同接近 20 次上限视为高风险，即使最终完成也需要分析原因。

## 6. 安全测试规则

- 不使用真实业务项目作为 Agent 工作区。
- 不把 `backend/config.json`、数据库、memory 中的真实内容贴进报告。
- 测试输出中出现 api_key、token、cookie 只记录是否存在、是否脱敏，不记录值。
- 路径穿越用 `../x`、绝对路径、非法 slug 验证，不尝试读取真实敏感文件。
- Kill/执行器测试必须使用临时目录，运行后检查无越界产物。

## 7. 发布准入

- P0 全绿。
- P1 无高危失败；中危失败需有明确规避方案。
- 协同专项评分 >= 80。
- 安全测试无高危问题。
- 若真实模型协同未执行，报告必须标注原因，不能把假执行器结果等同于真实智能质量。
