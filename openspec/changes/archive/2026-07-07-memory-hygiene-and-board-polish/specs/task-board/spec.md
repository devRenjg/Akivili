# task-board

## MODIFIED Requirements

### Requirement: 看板分列展示

系统 SHALL 按状态分列展示项目的任务，「验证中」为一等看板列，位于「进行中」与「已完成」之间；每列内的顶层任务 SHALL 按创建时间倒序排列（最新在最上）。

#### Scenario: 查看看板
- **WHEN** 用户进入项目工作区
- **THEN** 系统按 待办/进行中/验证中/已完成 分列展示任务；「验证中」独立成列（区别样式），归档任务单独可见

#### Scenario: 最新任务在最上
- **WHEN** 某列有多个顶层任务
- **THEN** 系统按创建时间倒序排列，最新创建的任务出现在该列最上方

#### Scenario: 卡片执行状态对齐
- **WHEN** 卡片底部展示执行状态（执行中/完成/失败/终止），无论该卡片是否带子任务进度
- **THEN** 执行状态固定右对齐，不因有无子任务进度而在左右间横跳

#### Scenario: 待人工验收
- **WHEN** 任务执行完成进入验证中
- **THEN** 该任务停留在「验证中」列，直到管理员手动拖入「已完成」完成验收
