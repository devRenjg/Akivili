# agent-reflection

## MODIFIED Requirements

### Requirement: 反思产出为 Know-how 而非结论存档

系统 SHALL 让每个角色用**自身模型与人格**复盘，产出少量（最多 3-5 条）聚焦「方法、坑、诀窍、判断依据」的可迁移经验，而非复述任务的结论或数据；SHALL 设质量门槛——只沉淀本专业领域的新增量，宁缺毋滥。

#### Scenario: 提炼经验
- **WHEN** 某角色对已完成任务复盘
- **THEN** 以其人格为背景、以其在本任务的产出为素材，产出数条一句话经验，写入其记忆的 Know-how 受管段落

#### Scenario: 低价值任务不灌套话
- **WHEN** 本次任务是常规沟通/介绍/自我描述/汇报/走流程类、无可迁移到未来工作的专业增量
- **THEN** 该角色反思只回「无」，不写放之四海皆准的通用套话（如"用固定结构表达""锚定项目场景"这类空话）

#### Scenario: 无可沉淀时不写
- **WHEN** 本次任务确无值得沉淀的新经验
- **THEN** 不向记忆写入任何内容
