# task-board

## Purpose

项目下的看板式任务管理。工作拆成任务，任务有负责人、有状态流转，可废弃归档，看板按状态分列展示。

## Requirements

### Requirement: 任务 CRUD 与负责人

用户 SHALL 能在项目下创建任务（标题、描述），并指定一个团队成员为负责人。

#### Scenario: 创建任务
- **WHEN** 用户在项目工作区新建任务并填写标题
- **THEN** 系统创建任务（默认状态 规划中）并建立其对话 Thread

#### Scenario: 指定负责人
- **WHEN** 用户为任务指定团队中的某成员
- **THEN** 系统记录该任务的负责人

### Requirement: 状态流转

任务状态 SHALL 在 规划中 → 进行中 → 验证中 → 已完成 间流转，且可随时废弃为已归档。

#### Scenario: 推进状态
- **WHEN** 用户改变任务状态
- **THEN** 系统保存新状态，看板中该任务移动到对应列

#### Scenario: 废弃归档
- **WHEN** 用户废弃某任务
- **THEN** 任务状态变为已归档，从活跃看板移除

### Requirement: 看板分列展示

系统 SHALL 按状态分列展示项目的任务。

#### Scenario: 查看看板
- **WHEN** 用户进入项目工作区
- **THEN** 系统按 规划中/进行中/验证中/已完成 分列展示任务，归档任务单独可见
